# gtts-dic
Словарь для связки GoogleTTS + TTSLexx

Описание:
Русский словарь для использования на устройствах Andoid для коррекции произношения движка Speech Services by Google (далее gtts). Используется только совместно с TTSLexx, которые могут быть установлены из маркета или других источников.

Чтобы скачать: нажмите на "Code:" (зеленая кнопка) и далее в меню выберите "Download ZIP"

Особенности:
1. Проверена работоспособность на версии движка gtts 24.9.361717975, на более новых версиях работает корректно. Однако, офф-лайн голоса в более новых версиях испорчены (хрипы и придыхания в конце фраз).
2. TTSLexx можно установить отсюда: https://play.google.com/store/apps/details?id=sia.netttsengine.ttslexx Поддерживаются 2 набора голосов: офф- и он-лайн. Словарь работает с обоими вариантами, но с он-лайн голосами тестирования не проводилось. Если предполается использовать оба набора голосов, то словарь нужно загружать дважды, т.е. для каждого набора отдельно - см. настройки TTS.
3. Словарь намеренно сделан таким образом, чтобы охватить максимальное количество словоформ, вне зависимости от того, произносится слово движком правильно или нет. Вызвано это тем, что на практике невозможно отловить все ошибки в обозримое время, а офф-лайновый gtts в части корректности произношения развивается медленно. В результате словарь получился большой, что очень положительно сказалось на качестве поизношения, но несколько пострадала производительность.
4. Все шаблоны словаря намеренно включают начало слова, это связано с особенностями работы TTSLexx. Предполагается, что ни одна словоформа не обрабатывается двумя шаблонами, хотя отдельные ошибки могут еще встречаться.
5. Шаблоны типа regex используются минимально и только для работы с омонимами. Они сильно снижают производительность и для других целей их использовать крайне не рекомендуется.
6. Наличие "лишних" букв в правой части шаблонов или "неправильное" написание слова призвано исправить произношение движка. Это не ошибка, если слово произносится верно. Второй функцией "лишних" букв является предотвращение срабатывания одного шаблона на результат работы другого.
Словарь наверняка содержит множество неизвестных еще ошибок, о которых желательно сообщать. Пишите, будем исправлять, но помните о наличие в языке омонимов.
7. Новые версии словаря будут периодически публиковаться по мере внесения изменений в него.

Особенности работы TTSLexx
Перед внесением изменений в словарь следует знать его особенности. Рекомендуется почитать инструкцию к нему и соответствующую тему на 4pda (название: Speech Services by Google (Синтезатор речи Google, ex. Google text-to-speech), где можно пообщаться с автором TTSLexx - ssiiaa. Но основное это:
1. Словоформа проходит через весь список шаблонов словаря, т.е. при первом срабатывании шаблона обработка не прекращается. Т.о. особое внимание уделяется исключению двойных срабатываний.
2. Порядок сортировки шаблонов в процессе работы: (1) шаблоны regex, (2) шаблоны типа "сло"="сло'", (3) шаблоны типа слово=сло'во.
3. Шаблоны (1) и (2) могут обрабатывать часть слова и, следовательно, могут быть причиной двойной обработки слова, коророе может получить 2 ударения и будет произноситься неправильно.
4. С практической точки зрения можно считать, что TTSLexx обрабатывает только буква русского алфавита и знак ударения. Прочие символы и буквы unicode либо игнорируются либо преобразуются в тот же стандартный русский алфавит.
5. regex-шаблоны работает в зависимости от того, как их понимает система. Тестирование производилось на Android 11. Использовать с осторожностью, если наблюдаются проблемы с производительностью, попробовать удалить эту секцию словаря в первую очередь.
